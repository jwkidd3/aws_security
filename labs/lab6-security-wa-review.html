Mini Well-Architected Security Review

Module 6 — Course Wrap-Up

Objective

Run the morning's seven Security Pillar design principles against a fictional workload. Score each as Meets / Partial / Gap, then turn the gaps into a prioritized remediation roadmap.

Terms You'll See

  • Well-Architected Review — structured walkthrough of a workload against AWS best practices
  • Workload — the collection of resources delivering business value
  • Finding — a best practice that isn't met (scored High / Medium / None risk)
  • Improvement plan — the tool's suggested services & docs for each finding
  • Milestone — a saved snapshot of review answers; compared on re-review

Scenario — "OrderFlow"

A 2-year-old e-commerce order-processing workload in one AWS account:

  • Runs in one region, one VPC, one account shared with two other apps
  • Public ALB → EC2 auto-scaling group → RDS MySQL (Multi-AZ)
  • Images & receipts in S3 (orderflow-assets, public-read for CloudFront)
  • Developers use IAM users with access keys; keys last rotated "a while ago"
  • CloudTrail enabled in us-east-1 only; logs go to the same account
  • Security Hub & GuardDuty are "on the roadmap"
  • Secrets are in a .env file baked into the AMI
  • TLS on the ALB; RDS doesn't enforce SSL; S3 bucket has default SSE-S3
  • No documented IR process; no game days ever run

Sample Answer — Scoring

  • Identity — Gap: IAM users + long-lived keys; no Identity Center; no MFA evidence
  • Traceability — Gap: CloudTrail single-region, no central log account, Security Hub/GuardDuty off
  • Layers — Partial: VPC + SGs exist, but S3 bucket public-read, no WAF
  • Automate — Gap: no Config rules, no SCPs, no policy-as-code
  • Protect data — Partial: ALB TLS yes; RDS no SSL enforcement; S3 SSE-S3 but no CMK for sensitive data
  • Keep people away — Gap: secrets in .env baked into AMI; devs have direct creds
  • Prepare for events — Gap: no IR playbook, no game days, no break-glass roles

Sample Answer — Top-5

  1. Enable CloudTrail in all regions and deliver to a dedicated log-archive account — visibility is the prerequisite for everything else
  2. Turn on GuardDuty + Security Hub org-wide; route High findings to Slack/ticket system
  3. Migrate developers off IAM user access keys to IAM Identity Center; enforce MFA
  4. Move secrets out of the AMI into Secrets Manager; add rotation
  5. Fix the public orderflow-assets bucket — CloudFront with OAC + Block Public Access on

Everything else (WAF, Config rules, IR playbooks) still matters — but these five give you the most risk reduction for the least effort, and most unlock the rest.

Console Exploration (read-only)

Navigate to these in the AWS Console. Do not create, edit, or save anything.

  1. AWS Well-Architected Tool → Define workload. Start the flow (fill in fictional values), but do not click Save

  3. Trusted Advisor → Security. Browse the checks (open SGs, MFA on root, IAM use)
