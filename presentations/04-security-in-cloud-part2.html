Security IN the Cloud — Part 2

Securing infrastructure · Monitoring & detective controls

🧅 Defense in Depth, Revisited

No single control is enough. Layer them.

  • Identity — MFA, least privilege, short-lived credentials
  • Network — VPCs, subnets, security groups, WAF
  • Host — patched AMIs, SSM Session Manager, anti-malware
  • Application — input validation, SCA, secrets management
  • Data — encryption at rest/in transit, tokenization
  • Detection — CloudTrail, GuardDuty, Security Hub

🌐 VPC — Your Private Network

  • VPC — a logically isolated network in AWS, spanning multiple AZs
  • Subnet — a slice of the VPC's IP range, tied to one AZ
  • Public subnet — routes to an Internet Gateway; reachable from the internet
  • Private subnet — no direct internet path; outbound via NAT Gateway if needed
  • Common pattern: ALB/NLB in public subnets → app in private subnets → DB in isolated subnets

🚧 Security Groups vs. NACLs

  • Security group — per-ENI firewall, stateful (return traffic auto-allowed), default-deny inbound
  • NACL (Network Access Control List) — per-subnet, stateless, numbered rules, default-allow — rarely needed beyond defaults
  • SG rules can reference other SGs — the cleanest way to express "app may reach DB"
  • Audit tip: any SG allowing 0.0.0.0/0 on a non-web port is worth a second look

🔗 Private Connectivity — No Internet Required

  • VPC endpoint — Interface (PrivateLink) — a private ENI in your subnet for an AWS service or SaaS
  • VPC endpoint — Gateway — S3 and DynamoDB only, added to a route table
  • Transit Gateway — hub-and-spoke routing between many VPCs and on-prem
  • Direct Connect / Site-to-Site VPN — private links back to corporate data centers
  • Keeping AWS-service traffic off the public internet is both a security and a cost win

🛡️ Web & Edge Protection

  • AWS WAF — L7 rules blocking SQLi, XSS, bad bots, rate abuse; attached to ALB, API Gateway, CloudFront, AppSync
  • AWS Shield Standard — free, on by default, L3/L4 DDoS protection at the edge
  • AWS Shield Advanced — paid; 24×7 DRT access, cost protection, L7 protection
  • CloudFront — CDN that absorbs volumetric traffic and terminates TLS near users
  • AWS Network Firewall — deep-inspection, domain-aware egress filtering

💪 Host & Workload Hardening

  • SSM Session Manager — browser-based shell into EC2 — no SSH keys, no open port 22, full audit log
  • Patch Manager — orchestrated OS and package patching on a cadence
  • EC2 Image Builder — golden AMIs built from a pipeline with hardening baked in
  • Inspector — continuous vulnerability scanning of EC2, ECR images, and Lambda functions
  • IMDSv2 required — the Instance Metadata Service hands out the EC2 instance's temporary IAM credentials; v1 answered any local HTTP GET (which SSRF bugs in web apps can trick the server into making), so attackers harvested the creds. v2 requires a session token first — close that whole class of attack by disabling v1.

📊 Monitoring — The Foundational Logs

  • CloudTrail — every control-plane API call in the account, persisted to S3; enable in all regions & all accounts, deliver to a central log account
  • VPC Flow Logs — IP flow metadata for ENIs (accept/reject, src/dst, bytes)
  • DNS (Route 53 Resolver) query logs — names being looked up from your VPC
  • CloudWatch Logs — OS and application logs, queryable with Logs Insights
  • S3 server access / data-event logs — per-object reads and writes

🔍 Detective Controls

Detective controls find problems after they happen so you can respond.

  • GuardDuty — threat detection on CloudTrail, VPC Flow, DNS, EKS, S3, Malware, RDS — no agents
  • Security Hub — aggregates findings from GuardDuty, Inspector, Macie, Config, partners; scores against CIS, PCI, AWS FSBP
  • AWS Config — continuous resource inventory + rules ("no unencrypted EBS," "MFA on root")
  • Detective — graph-based investigation across CloudTrail & VPC flows

⚡ Turning Detection into Response

  • EventBridge — route a GuardDuty or Security Hub finding to a Lambda, SSM runbook, or ticket
  • Auto-remediation — Config rule sees a public S3 bucket → Lambda blocks public access automatically
  • Delegated admin — a central security account owns Security Hub / GuardDuty across the org
  • Practice the path: finding → alert → action. An alert nobody acts on is noise.
