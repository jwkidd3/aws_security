AWS Well-Architected Framework — Recap

  • A published set of best practices across six pillars: Operational Excellence, Security, Reliability, Performance, Cost, Sustainability
  • Everything we covered today maps to the Security Pillar questions
  • Free PDF whitepaper — a living document AWS updates as services evolve
  • The Well-Architected Tool in the console is the interactive version

The Well-Architected Tool

  • Free AWS Console service
  • Define a workload (your app/system) and answer the questionnaire pillar by pillar
  • Each answer maps to best practices; gaps become findings scored High / Medium / None
  • Each finding links to an improvement plan (AWS docs, services, patterns)
  • Save a milestone (snapshot) and re-review later to track progress

A Typical Security Review Flow

  1. Architect + security owner + workload team meet for 60–90 min
  2. Walk the Security Pillar questions honestly
  3. Tool produces a risk report
  4. Group findings by theme, assign owners, set target dates
  5. Add remediation items to the normal engineering backlog
  6. Re-run the review quarterly or at major milestones

Security Best-Practice Checklist (Today in One Slide)

  • Organizations + SCPs + Control Tower for the org-level guardrails
  • Root locked with hardware MFA, no access keys
  • IAM Identity Center for humans; IAM roles for workloads; no long-lived keys
  • CloudTrail in all regions, delivered to a central log account
  • Config + Config rules + Security Hub standards enabled
  • GuardDuty & Inspector enabled org-wide
  • S3 Block Public Access on; default encryption; versioning; Object Lock on regulated data
  • KMS customer-managed keys for sensitive data; Secrets Manager for credentials
  • CloudFront + WAF + Shield in front of public apps
  • Private subnets for app + DB; VPC endpoints for AWS services; no 0.0.0.0/0 SGs on app ports
  • Written IR runbook, tested via game day in the last quarter

Anti-Patterns to Watch For

  • "It's behind a VPN, so it's fine" — identity and data controls still apply
  • "We'll enable CloudTrail later" — later is too late
  • "Everyone gets AdministratorAccess for now"
  • "We rotate keys manually when we remember"
  • "Backups exist" — but have never been restored
  • "GuardDuty fires alerts but nobody reads them"
  • "We haven't had an incident, so IR can wait"

Take-Home Challenge

Pick one workload you own. In the next two weeks:

  1. Run it through the Well-Architected Security Pillar in the console
  2. Pick the top three findings with the biggest risk-reduction payoff
  3. Write them up as engineering tickets with owners and target dates
  4. Schedule a re-review 90 days out

That's all it takes to start a real security program — and you already have the template.
