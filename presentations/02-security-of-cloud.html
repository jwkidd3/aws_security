Security OF the Cloud

What AWS handles for you — and the evidence you can inherit.

🌍 AWS Global Infrastructure

  • Region — a geographic area (e.g., us-east-1); data stays in-region unless you move it
  • Availability Zone (AZ) — one or more discrete data centers in a region, isolated from other AZs
  • Edge location — Points-of-Presence used by CloudFront and the global network
  • Local Zones / Wavelength / Outposts — AWS infrastructure closer to users or on-prem
  • Security, latency, and compliance all drive where you deploy

🏢 Data Center Security

Controls AWS runs on your behalf — you don't configure these, you inherit them.

  • Physical — perimeter controls, badge access, 24×7 guards, CCTV, disk destruction logs
  • Environmental — fire suppression, power redundancy, climate control
  • Personnel — background checks, need-to-know access, separation of duties
  • Hardware & Nitro — purpose-built hypervisor with a hardware security root of trust
  • Decommissioning — NIST 800-88 media sanitization before hardware leaves the floor

🌐 Network Security at the Infrastructure Layer

  • All traffic between AZs travels over AWS's own fiber, not the public internet
  • All data in transit between Regions is encrypted by default on the AWS backbone
  • DDoS protection is on by default at the edge — Shield Standard is built-in and free
  • The Nitro System isolates your workload from AWS operator access — even AWS staff can't read your running instance memory

📋 Compliance & Governance

AWS maintains dozens of third-party attestations. You inherit the infrastructure controls.

  • SOC 1 / SOC 2 / SOC 3 — financial & trust-services controls
  • ISO 27001 / 27017 / 27018 — information-security and cloud-specific standards
  • PCI-DSS — payment-card industry
  • HIPAA / HITRUST — U.S. healthcare
  • FedRAMP / DoD SRG — U.S. government
  • GDPR — EU data-protection readiness tooling

📄 AWS Artifact

  • Self-serve portal in the AWS Console for downloading AWS compliance reports
  • Download SOC reports, ISO certificates, PCI AOC, FedRAMP packages, and more
  • Sign AWS-offered agreements (BAA for HIPAA, NDAs) in-place
  • Auditors asking "show me the SOC 2"? That's Artifact

🏛️ Governance Services — Your Side

AWS provides the evidence; you provide the organization-wide policy.

  • AWS Organizations — multi-account management under a single billing root
  • Service Control Policies (SCPs) — account-wide guardrails; even an admin can't exceed them
  • Control Tower — opinionated landing zone with built-in guardrails
  • AWS Config — continuous configuration tracking and rule-based compliance
  • Audit Manager — continuously collects evidence mapped to frameworks (PCI, HIPAA, CIS…)

🗂️ Multi-Account Strategy (Quick Tour)

  • Separate accounts are the strongest AWS isolation boundary — blast radius, billing, and audit scope all shrink
  • Common structure: management, log archive, security tooling, prod, non-prod, sandbox OUs
  • SCPs applied at the OU level enforce region allow-lists, deny-root-user, prohibit public S3, etc.
  • Central logs and security findings flow into dedicated security accounts

✅ Key Takeaways

  • AWS secures the physical, network, and virtualization layers — you inherit those controls with evidence in Artifact
  • You still own configuration, identity, data, and operations in your accounts
  • Governance tools (Organizations, SCPs, Config, Control Tower) are where your side of the line becomes enforceable
